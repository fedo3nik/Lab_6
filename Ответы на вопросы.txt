1.Каким образом можно создать поток?
	Для создания нового потока мы можем создать новый класс, либо наследуя его от класса Thread, либо реализуя в классе интерфейс Runnable.
	Реализуется метод run()
2.В каких состояниях может прибывать поток?
	1)New
	2)Runnable
	3)Running
	4)Blocked or Waiting
	5)Dead
3.Можно ли запустить поток дважды?
	Метод start() у объекта Thread можно вызвать лишь единожды.
4.Для чего нужны start() и run()?
	Метод start() - это метод класса Thread который создает отдельный поток и вызывает в его контексте метод run(), 
	объекта на основании которого бы создан объект класса Thread.
5.Чем отличаются методы interrupt, interrupted, isInterrupted?  
 	Метод interrupt(). Вызов этого метода устанавливает у потока статус, что он прерван. 
	Сам метод возвращает true, если поток может быть прерван, в ином случае возвращается false.
	Методы interrupted() и isInterrupted() проверяют прерван ли поток, но первый снимает флаг прерывания,
	а второй нет, а также метод interrupted() статический.
6.Что   такое монитор объекта? Как   работают   методы   wait   и notify/notifyAll?
	Монитор — это дополнительная «надстройка» над мьютексом.
	Мьютекс — это специальный объект для синхронизации потоков.
	Монитор создает защитный механизм например для ключевого слова synchronized, захватывая мьютекс объектом.
	1)Метод wait()
	У метода wait() есть три вариации. Один метод wait() бесконечно ждет другой поток, пока не будет вызван метод notify() 
	или notifyAll() на объекте. Другие две вариации метода wait() ставят текущий поток в ожидание на определенное время. 
	По истечении этого времени поток просыпается и продолжает работу.
	2)Метод notify()
	Вызов метод notify() пробуждает только один поток, после чего этот поток начинает выполнение. Если объект ожидают 
	несколько потоков, то метод notify() разбудит только один из них. Выбор потока зависит от системной реализации управления потоками.
	3)Метод notifyAll()
	Метод notifyAll() пробуждает все потоки, хотя в какой последовательности они будут пробуждаться зависит от реализации ОС.
